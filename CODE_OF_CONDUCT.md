# Code of Conduct

TeleFlow is a technical project. Discussions should stay professional, specific, and useful for improving the framework.

## Expected Behavior

- Be respectful when disagreeing.
- Critique code, APIs, behavior, and documentation instead of people.
- Keep issue and pull request discussions focused on reproducible facts, tradeoffs, and maintainable solutions.
- Assume public comments may be read by new users, contributors, and organizations evaluating the project.
- Help keep the repository welcoming to contributors with different levels of experience.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or sustained hostility.
- Insults based on identity, background, nationality, language, gender, religion, disability, or experience level.
- Publishing private information, credentials, tokens, private chat data, or private production logs.
- Spamming issues, pull requests, discussions, or maintainers.
- Repeatedly derailing technical discussions after a maintainer asks to stop.

## Enforcement

Maintainers may edit, hide, lock, or delete comments and issues that violate this code of conduct. Maintainers may also block users from the repository or organization when behavior harms the project or its contributors.

Do not open public issues for conduct reports that include private information. Use GitHub's reporting tools or contact a maintainer through an appropriate private channel.

## Scope

This code of conduct applies to all project spaces, including issues, pull requests, discussions, documentation, examples, and official project communication channels.
